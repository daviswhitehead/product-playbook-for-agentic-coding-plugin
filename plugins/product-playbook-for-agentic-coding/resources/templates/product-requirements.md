### Project Size Guidelines
Project size determines the complexity and depth of planning required:

**Small Projects**:
- Simple, focused features or bug fixes
- 1-3 components or services
- Limited external dependencies
- Straightforward technical approach
- Estimated timeline: Days to 1-2 weeks

**Medium Projects**:
- Multiple components or features
- Some external integrations
- Moderate technical complexity
- Cross-cutting concerns (auth, data, UI)
- Estimated timeline: 2-4 weeks to 2 months

**Large Projects**:
- Complex systems with many components
- Multiple external integrations
- Significant technical challenges
- Multiple subsystems or services
- Estimated timeline: 2+ months
